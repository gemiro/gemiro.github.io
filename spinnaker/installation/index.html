Halyard 설치 및 Kubernetes에 Spinnaker 배포

2019/04/14

소개

Halyard는 Spinnaker deployment의 라이프사이클을 관리하기 위한 관리자 툴이다. Halyard를 통해 관리자는 Spinnaker를 배포 및 업데이트를 할 수 있으며, 또한 다양한 외부 Provider를 연동하여 기능을 확장 할 수 있다.

본 문에서는 Halyard를 Docker(Centos7 구성)로 설치하고 Kubernetes환경에 Spinnaker를 배포하는 방법을 소개한다.

사전 준비

  • Halyard를 설치하기 위한 Docker가 구성된 PC
  • 쿠버네티스 설치하기
  • Minio 분산 오브젝트 스토리지 구축

Halyard 설치하기

  1. Docker CE 가 설치된 PC를 준비한다.

  2. 그리고 Halyard config 디렉토리로 사용할 디렉토리를 만든다.

mkdir ~/.hal

.hal에 구성된 config파일은 Halayrd docker container에 Persist volume으로 구성된다.

  1. Halyard는 Spinnaker를 kubernetes에 배포할 때 kubectl 의 권한을 사용하여 배포한다. 따라서 halayrd에게 Kubernetes의 .kube 정보를 복사해 주어야 한다.

kubernetes가 설치된 master node에서 ‘~/.kube/config’ 파일을 복사하여 Halayrd를 설치할 PC로 복사한다. 그리고 아래 디렉토리를 생성하고 복사한 config 파일을 k8s폴더에 복사한다.

mkdir ~/.hal/k8s

  1. 다음 명령을 통해 Halyard docker container를 띄운다.
 docker run  \
   --name halyard --rm  \
   -v ~/.hal:/home/spinnaker/.hal \
   -v ~/.hal/k8s:/home/spinnaker/.kube \
   -d gcr.io/spinnaker-marketplace/halyard:stable
  1. Halyard가 정상적으로 올라갔으면 다음 명령어를 통해 Halyard로 연결한다.

docker exec -it halyard bash

  1. 다음 명령어는 command 완성 가이드를 활성화 한다. (편의를 위해)

source <(hal --print-bash-completion)

hal 명령어는 help 정보를 보여준다. Halyard Command Reference에서 명령어 정보를 확인할 수 있다.

Cloud Providers 선택하기

어플리케이션을 어떤 플랫폼으로 배포할지를 선태해야 한다. Spinnaker는 다양한 Providers를 지원한다. 본 문에서는 Kubernetes v2(manifest based)를 선택하여 환경을 구성한다.

  1. 다음 명령어를 통해 kubernetes provider를 활성화 한다.

hal config provider kubernetes enable

  1. 그 다음 kubernetes account를 생성한다.
CONTEXT=$(kubectl config current-context)
K8SACCOUNT=my-k8s-v2-account 
#my-k8s-v2-account 계정 명 (변경 가능)
hal config provider kubernetes account add $K8SACCOUNT \
    --provider-version v2 \
    --context $CONTEXT
    
hal config features edit --artifacts true

환경 선택

Halyard는 3가지 형태로 Spinnaker를 배포할 환경을 선택할 수 있다. Kubernetes에 배포하기 때문에 선택은 Distributed installation 밖에 없다.

다음 명령어를 통해 k8s account가 어떤 형태로 spinnaker를 배포시킬 것인지를 설정한다.

hal config deploy edit --type distributed --account-name $K8SACCOUNT

Persistent 외부 스토리지 설정

Spinnaker는 어플리케이션과 파이프라인의 설정 정보를 유지하기 위해 외부 스토리지 provider를 요구한다.

Spinnaker에 설정할 수 있는 외부 스토리지를 보면 Public cloud 방식의 외부 스토리지를 지원한다. 간단하게 local에 구축한다면 Redis를 선택할 수 있겠지만 추천하지 않는다고 하기 때문에 만약 local 환경만 갖고 구축한다면 Minio를 추천한다.

Minio 분산 오브젝트 스토리지 구축을 통해 Minio가 설치가 되었다면 Minio에 접근할 http://ip:9000 형태의 ENDPOINT와 MINIO_ACCESS_KEY, MINIO_SECRET_KEY가 준비되었을 것이다.

이 3개의 설정 정보가 있어야 Spinnaker에서 외부 스토리지 설정이 가능하다.

Halyard Command Reference를 보면 실제로 Minio 설정과 관련된 내용이 없다. 이는 Minio가 S3-호환이기 때문에 S3 외부 스토리지 설정을 그대로 사용하기 때문이다.

하지만 S3 설정을 그대로 사용할 수 없고 한가지 수정이 필요하다. Minio는 versioning object를 지원하지 않기 때문에 ~/.hal/$DEPLOYMENT/profiles/front50-local.yml 파일에 다음 한줄을 추가해야 한다.

spinnaker.s3.versioning: false

$DEPLOYMENT는 보통 default이다. 만약 ront50-local.yaml파일이 존재하지 않으면 파일을 생성시켜 준다

아래 명령어를 통해 s3 storage를 설정한다.

echo $MINIO_SECRET_KEY | hal config storage s3 edit --endpoint $ENDPOINT \
    --access-key-id $MINIO_ACCESS_KEY \
    --secret-access-key # will be read on STDIN to avoid polluting your
                        # ~/.bash_history with a secret
                        
hal config storage edit --type s3

Spinnaker 배포

  1. 버전 선택

다음 명령어는 사용 가능한 Spinnaker version list를 보여준다.

hal version list

리스트 중 하나를 다음 명령어로 선택하여 설정한다.

hal config version edit --version $VERSION

  1. Spinnaker 배포

hal deploy apply

Spinnaker Connect UI

배포된 Spinnaker UI에 접근하기 위해서는 배포된 Kubernetes Service 부분을 edit 해주어야 한다.

접속 방법은 NodePort를 이용하거나 LoadBalancer를 이용하는 방법이 있는데 본 문에서는 LoadBalancer를 이용한 방법을 설명한다.

  1. Kubernetes Master node에서 kubectl -n spinnaker get all 명령어를 통해 Spinnaker가 잘 배포 되었는지 먼저 확인한다.

확인 하였으면 다음 명령어를 통해 spin-deck service를 수정한다. kubectl edit svc spin-deck -n spinnaker

...
spec:
  clusterIP: 10.109.5.185
  externalTrafficPolicy: Cluster
  ports:
  - nodePort: 30530
    port: 80 # 9000 --> 80
    protocol: TCP
    targetPort: 9000
  selector:
    app: spin
    cluster: spin-deck
  sessionAffinity: None
  type: LoadBalancer # ClusterIP --> LoadBalancer
...

할당된 IP를 확인한다. $UI_IPADDRESS

그 다음 kubectl edit svc spin-gate -n spinnaker를 통해 spin-gate를 수정ㅎ나다.

...
spec:
  clusterIP: 10.99.188.44
  externalTrafficPolicy: Cluster
  ports:
  - nodePort: 30335
    port: 80  # 9000 --> 80
    protocol: TCP
    targetPort: 8084
  selector:
    app: spin
    cluster: spin-gate
  sessionAffinity: None
  type: LoadBalancer  # ClusterIP --> LoadBalancer

...

할당된 IP를 확인한다. $API_IPADDRESS

  1. halyard에서 ui 접근과 api 접근에 대한 base-url를 변경한다.

hal config security ui edit --override-base-url http://$UI_IPADDRESS

hal config security api edit --override-base-url http://$API_IPADDRESS

마지막으로 Spinnaker를 재 배포한다.

hal deploy apply

적용이 완료 되었으면 브라우저에서 http://$UI_IPADDRESS 으로 접속하여 Spinnaker UI가 잘 동작하는지 확인한다.
